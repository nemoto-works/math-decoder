# CHANGELOG

## 記録形式

各リリースまたは主要な変更単位で、必要に応じて以下の区分を使います。

- **Added**: 新機能や新しい学習コンテンツ
- **Changed**: 既存機能、表示、レイアウト、学習体験の変更
- **Fixed**: 不具合修正やデグレ解消
- **Known Issues**: 既知の課題、未解決の制約、今後確認が必要な点
- **Related**: 関連する Issue / PR
